📋 Spec機能の威力

要件定義フェーズ

## 要件 1: アカウント登録

**ユーザーストーリー:** エンジニアとして、
アカウントを作成したいので、プロフィールを
作成して管理できるようにしたい。

#### 受け入れ基準
1. WHEN ユーザーが登録ページにアクセスする 
   THEN Linker SHALL メールアドレスと
   パスワードの入力フォームを表示する

設計フェーズ

interface Profile {
  id: string;
  user_id: string;
  name: string;
  jobTitle: string;
  bio?: string;
  imageUrl?: string;
  skills: string[];
  yearsOfExperience?: number;
  socialLinks: SocialLink[];
  createdAt: string;
  updatedAt: string;
}

ポイント: 要件→設計→実装の一貫性をKiroが保証

🧪 プロパティベーステストの革新

従来のテスト

// 特定のケースをテスト
test('プロフィール作成', () => {
  const profile = createProfile({
    name: "田中太郎",
    jobTitle: "エンジニア"
  });
  expect(profile.name).toBe("田中太郎");
});

❌ 限定的なケースのみ

プロパティベーステスト

// 普遍的な性質をテスト
property('プロフィール作成の一貫性', 
  fc.record({
    name: fc.string(),
    jobTitle: fc.string()
  }), (data) => {
    const profile = createProfile(data);
    return profile.name === data.name &&
           profile.jobTitle === data.jobTitle;
  });
