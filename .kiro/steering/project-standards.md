# プロジェクト標準

このドキュメントは、linkerプロジェクトの開発標準とベストプラクティスを定義します。

## 言語設定

### コミュニケーション言語
- **すべてのやり取りは日本語で行う**
- AIアシスタントとのコミュニケーションは日本語を使用する
- 新しいセッションが開始された場合でも、日本語でやり取りを継続する
- コードコメントやドキュメントも日本語で記述する（変数名・関数名は英語）

## コーディング規約

### 一般原則
- コードは読みやすく、保守しやすいものにする
- 明確で説明的な変数名と関数名を使用する
- 複雑なロジックにはコメントを追加する
- DRY原則（Don't Repeat Yourself）に従う

### ファイル構成
- 関連する機能は同じディレクトリにまとめる
- テストファイルはソースファイルと同じディレクトリに配置する
- 明確なディレクトリ構造を維持する

## テスト戦略

### テストの種類
- **ユニットテスト**: 個別の関数やクラスの動作を検証
- **プロパティベーステスト**: 普遍的な性質を多数の入力で検証
- **統合テスト**: コンポーネント間の連携を検証

### テストの原則
- すべての新機能にはテストを書く
- テストは実際の機能を検証する（モックに頼りすぎない）
- エッジケースと正常系の両方をカバーする
- テストは独立して実行可能にする

### テストコマンド
- **テスト実行**: `npm test` (vitest --runを実行)
- **ウォッチモード**: `npm run test:watch` (変更を監視して自動実行)
- **UIモード**: `npm run test:ui` (ブラウザでテスト結果を表示)

## エラーハンドリング

- 適切なエラーメッセージを提供する
- エラーは適切なレベルでキャッチする
- ユーザーフレンドリーなエラーメッセージを心がける

## ドキュメント

- 公開APIには必ずドキュメントを書く
- 複雑なアルゴリズムには説明を追加する
- READMEは常に最新の状態に保つ

## Git運用ルール

### コミットとプッシュ
- タスク完了ごとに必ずGitHubへプッシュする
- コミットメッセージは日本語で明確に記述する
- コミットメッセージの形式:
  ```
  feat: タスク名
  
  - タスクX: 実装内容の説明
    - 詳細1
    - 詳細2
  ```
- プッシュ手順:
  1. `git add .`
  2. `git commit -m "メッセージ"`
  3. `git push origin main`
