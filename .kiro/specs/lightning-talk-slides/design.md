# 設計書

## 概要

「Kiroでアプリを作ってみた。」ライトニングトークは、AWS re:Invent2025での体験から始まり、AI支援開発ツールKiroを使用してLinkerプラットフォームを開発した実体験を共有するプレゼンテーションです。10-15分の発表時間で、技術者と非技術者の両方に理解しやすい内容を提供し、AI支援開発の実際の効果と可能性を伝えます。

## アーキテクチャ

### プレゼンテーション構造

```
ライトニングトーク構成:
1. タイトルスライド (1分)
2. 自己紹介 (1-2分)
3. AWS re:Invent 2025 体験 (2-3分)
4. Kiro紹介 (2-3分)
5. 開発プロセス実演 (3-4分)
6. 成果とデモ (3-4分)
7. 学びと展望 (2-3分)
8. まとめ・質疑応答 (1-2分)
```

### 技術スタック

**スライドシステム:**
- HTML5 + CSS3 + JavaScript (ES6+)
- Reveal.js フレームワーク
- Prism.js (シンタックスハイライト)
- レスポンシブデザイン

**コンテンツ管理:**
- Markdown形式でのスライド記述
- 画像・スクリーンショット管理
- コードスニペット埋め込み

**プロジェクト構造:**
```
workspace/
├── src/                       # Linkerアプリケーション（既存・そのまま）
├── package.json               # Linkerの設定（既存・そのまま）
├── ... (既存のLinkerファイル)
├── slides/                    # ライトニングトークスライド（新規）
│   ├── index.html
│   ├── css/
│   ├── js/
│   ├── images/
│   └── content/
└── .kiro/specs/lightning-talk-slides/  # スライドのスペック
```

## テスト戦略

### 二重テストアプローチ

**ユニットテスト:**
- 個別スライドコンテンツの正確性検証
- リンクの有効性確認
- 画像・メディアファイルの存在確認
- 技術情報の事実確認

**プロパティベーステスト:**
- fast-checkライブラリを使用
- 最小100回の反復実行
- 各プロパティベーステストに設計書のプロパティ参照コメントを付与
- フォーマット: `**Feature: lightning-talk-slides, Property {number}: {property_text}**`

**統合テスト:**
- プレゼンテーション全体の流れ確認
- 時間配分の妥当性検証
- デモシナリオの実行可能性確認

### テスト実装要件

- プロパティベーステストライブラリ: fast-check
- 各プロパティベーステストは最小100回実行
- 各テストは対応する正確性プロパティを明示的に参照
- ユニットテストと プロパティテストの両方を実装して包括的カバレッジを提供

## パフォーマンス要件

### プレゼンテーション実行

- スライド切り替え: 500ms以内
